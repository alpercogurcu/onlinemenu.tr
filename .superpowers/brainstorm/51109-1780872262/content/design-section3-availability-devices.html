Ürün müsaitlik modeli · Auto-close kuralı · Cihaz rol sistemi

Ürün Müsaitlik — Tablolar
products (catalog modülü)
id UUID PK
branch_id UUID
... mevcut alanlar ...
auto_close_on_zero_stock BOOL DEFAULT false
Şube sahibi/müdürü bu ayarı açarsa stok 0 olunca sistem otomatik kapatır. Çoğu şubede kapalı kalır.
product_channel_availability (yeni tablo)
product_id UUID
branch_id UUID
channel TEXT -- 'pos'|'delivery'|'yemeksepeti'|...
is_available BOOL DEFAULT true
changed_by UUID -- kim kapattı (audit)
changed_at TIMESTAMPTZ
PRIMARY KEY (product_id, branch_id, channel)
Öncelik Sırası — Kim Kazanır?
1
Manuel kapatma is_available = false (product_channel_availability) HER ZAMAN KAZANIR
2
Auto-close (stok = 0) auto_close_on_zero_stock = true AND stock ≤ 0 Manuel açık yazarsa ezilir
3
Varsayılan kayıt yok = açık AÇIK
Tablo satırı sadece kapalı veya override edilmiş ürünler için yazılır — açık ürünler için kayıt yok. Bu "closed set" yaklaşımı insert sayısını minimumda tutar.

Device Role Sistemi — MVP
-- devices tablosu (SEC-004 pairing'e bağlı)
id UUID PK
tenant_id UUID NOT NULL
branch_id UUID NOT NULL
name TEXT NOT NULL -- "Kasa 1", "Mutfak Ekranı"
roles TEXT[] NOT NULL -- ['kasa'] | ['garson'] | ['mutfak'] | ['kasa','garson']
device_type TEXT NOT NULL -- 'tablet' | 'kds_screen' | 'desktop' | 'pos_terminal'
is_active BOOL DEFAULT true
last_seen_at TIMESTAMPTZ -- heartbeat
-- pairing alanları SEC-004'ten: fingerprint, token, paired_at ...

WebSocket Kanal Routing — Role-based
delivery.order.received.v1 ──▶ ws/branch/{id}/role/kasa
└──▶ Kasa 1 ✓ alır   Kasa 2 ✓ alır
└──▶ Garson Tableti ✗ almaz   Mutfak Ekranı ✗ almaz

pos.order.created.v1 ──▶ ws/branch/{id}/role/mutfak
└──▶ Mutfak Ekranı ✓ alır (kanal bilgisiyle)

catalog.availability.changed.v1 ──▶ ws/branch/{id}/role/kasa + role/garson
└──▶ Tüm cihazlar menü refresh alır
