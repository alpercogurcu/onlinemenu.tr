Order Channel Mimarisi

Fake masa yok. Her sipariş nereden geldiğini bilir. Muhasebe ve royalty hesabı buna göre yapılır.

Sipariş Veri Modeli — orders tablosu
id UUID
tenant_id UUID -- RLS
branch_id UUID
table_id UUID | NULL -- sadece dine_in
order_channel TEXT NOT NULL -- dine_in | takeaway | click_collect | delivery
delivery_integrator_id UUID | NULL -- FK → branch_delivery_integrators (hangi platform)
external_order_id TEXT | NULL -- Yemeksepeti/Getir'in kendi order ID'si
delivery_address JSONB | NULL-- sadece delivery kanalı için
status TEXT -- open | preparing | ready | delivered | cancelled
total_amount NUMERIC
channel_commission NUMERIC -- Yemeksepeti'nin kestiği komisyon (raporlama)
4 Sipariş Kanalı — masa oluşturmak yok
🪑 dine_in
Masada oturan müşteri. table_id dolu. Normal adisyon akışı.
🛍️ takeaway
Gel-al (kasadan). Masa yok. delivery_integrator_id = null.
📱 click_collect
Online ödenmiş, gelip alacak. İskonto kuralı buraya uygulanır.
🛵 delivery
Yemeksepeti / Getir / Trendyol. delivery_integrator_id → hangi platform belli.

Royalty Kural Motoru — şube/franchise bazında config
franchise_royalty_rules JSONB (branch_settings içinde)
channel = dine_in → royalty_base'e dahil, tam tutar
channel = takeaway → royalty_base'e dahil, tam tutar
channel = click_collect → royalty_base'e dahil, -10% iskonto uygulanır
channel = delivery → royalty_base'den hariç (platform komisyon var)
💡 Her franchise şubesi farklı kural seti taşıyabilir. Kurallar branch_settings.royalty_rules JSONB içinde. Değişiklik → NATS event → royalty hesap servisi cache'i invalidate eder.
